Plan para aprobacion - UADE StartUP

Resumen ejecutivo del MVP, decisiones cerradas, riesgos y entregables necesarios para aprobar el inicio del desarrollo.

Resumen ejecutivo

UADE StartUP sera una plataforma web para mostrar proyectos por edicion, permitir votacion publica en tiempo real y registrar evaluaciones privadas de jurados. El producto queda preparado para crecer por ediciones, por ejemplo Edicion 2026, y conservar historicos consultables.

Propuesta MVP: una app publica para ver proyectos y ranking en vivo, Microsoft OAuth / Entra ID con correo UADE solo para votar, paneles para expositores/jurados/admin, backend Express, frontend Next.js, deploy en CapRover y archivos en MinIO.

Alcance MVP

Publico e invitados

  • Acceso por QR publico a la edicion activa.
  • Ver proyectos, detalles y ranking sin login.
  • Microsoft OAuth / Entra ID con correo UADE solo al votar.
  • Un voto activo por invitado y edicion, con posibilidad de cambiarlo.

Expositores

  • Alta gestionada por administrador.
  • Carga y edicion de proyecto.
  • Logo, imagenes, links y descripcion del proyecto.
  • Envio a revision/publicacion.

Jurados

  • Login interno.
  • Evaluacion con 1 a 5 estrellas.
  • Comentario privado opcional.
  • Edicion de calificacion hasta cierre de evaluacion.
  • Todos los jurados evaluan todos los proyectos de la edicion.

Administracion

  • Gestion de ediciones.
  • Gestion de usuarios, roles y equipos.
  • Moderacion de proyectos.
  • Resultados publicos y ranking de jurado privado.

Decisiones cerradas

AreaDecision
FrontendNext.js.
BackendNode.js con Express.
InfraestructuraCapRover con frontend y backend como apps separadas. El dominio final queda pendiente; se usara el patron nombreapp.dominio.com.
ArchivosMinIO autohosteado dentro de CapRover, con volumen persistente y backups.
Base de datosPostgreSQL dentro de CapRover.
VotacionRanking publico en tiempo real por WebSockets.
Login invitadoNo requiere login para mirar; Microsoft OAuth / Entra ID con correo UADE solo para votar.
Ranking publicoOrden por mayor cantidad de votos; empate por quien llego antes a esa cantidad.
Ranking juradoPromedio de estrellas, visible solo para administradores.
Evaluacion juradoTodos los jurados evaluan todos los proyectos de la edicion.
Ventana de votacionConfigurable por el administrador en cada edicion.
HistoricoSoporte por ediciones/versiones desde el modelo inicial.

Fuera del MVP

  • Notificaciones por email.
  • Asignacion avanzada de jurados.
  • Reportes PDF.
  • Comentarios publicos.
  • App mobile nativa.
  • Analitica avanzada de comportamiento.

Riesgos y mitigaciones

RiesgoImpactoMitigacion
Disponibilidad del servidor durante la muestraAltoPrueba de carga previa, monitoreo basico, reinicio controlado de apps y backups.
MinIO autohosteado sin backupsAltoVolumen persistente, backup programado y prueba de restauracion antes del evento.
Problemas con Microsoft OAuth / Entra ID el dia del eventoMedioConfigurar credenciales y dominio final con anticipacion; probar desde celulares y redes distintas.
Ranking en tiempo real con muchos usuariosMedioWebSockets con eventos chicos, contador desnormalizado y prueba de concurrencia.
Datos de jurado expuestos por errorAltoAutorizacion por rol en backend y pruebas especificas de permisos.

Faltantes antes de aprobar desarrollo

  • Confirmar dominio final. Mientras tanto, planificar subdominios con el patron CapRover nombreapp.dominio.com, por ejemplo frontend, backend y MinIO.
  • Confirmar datos obligatorios finales del formulario de proyecto. Propuesta inicial: titulo, resumen, descripcion, problema, solucion, categoria, etapa, integrantes, logo y contacto del equipo.
  • Definir fecha estimada de la Edicion 2026.
  • Preparar credenciales Microsoft OAuth / Entra ID para ambiente de desarrollo y produccion.
