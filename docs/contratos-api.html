Contratos de API - UADE StartUP

Contratos REST + WebSocket para el MVP. Los endpoints de auth interna, ediciones, admin y proyectos reflejan el backend Express actual; votacion, jurado, archivos y realtime quedan como contratos planificados.

Convenciones

Base URL actual/. El backend actual expone rutas sin prefijo /api/v1.
FormatoJSON.
Auth internaSesion HTTP-only o Bearer token para admin, jurado y expositor.
Auth invitadosNo se requiere login para ver ediciones, proyectos y ranking publico. Google OAuth solo es obligatorio para votar.
Errores{ "error": { "code": "string", "message": "string", "details": {} } }

Healthcheck

GET /health

Verifica que la API este levantada.

{
  "status": "ok",
  "service": "uade-startup-backend"
}

Auth

POST /auth/login

Login para usuarios internos: admin, jurado y expositor.

{
  "email": "admin@uade.edu.ar",
  "password": "********"
}
{
  "token": "jwt",
  "user": {
    "id": "uuid",
    "name": "Admin",
    "email": "admin@uade.edu.ar",
    "role": "admin"
  }
}

GET /auth/google

Planificado. Redirecciona al flujo OAuth Google. Se dispara cuando un invitado intenta votar sin estar autenticado.

GET /auth/google/callback

Planificado. Callback OAuth. Crea o recupera usuario invitado, valida que use cuenta Google y lo deja autenticado.

GET /auth/me

Devuelve usuario actual. Requiere Bearer token.

{

    "email": "admin@uadestartup.local",
    "role": "admin"
  }
}

GET /editions/:slug

Detalle de una edicion. No requiere login.

GET /editions/:slug/projects

Proyectos publicados de una edicion, ordenados por ranking publico si la edicion esta activa. No requiere login.

Edita proyecto. Rol: expositor propietario o admin.

Votos

POST /projects/:id/vote

Emite o cambia el voto activo del invitado autenticado con Google para la edicion del proyecto. Requiere Google Login.

{
  "previousProjectId": "uuid-or-null",
  "currentProjectId": "uuid",
  "ranking": [
    { "projectId": "uuid", "voteCount": 28, "position": 1 },
    { "projectId": "uuid", "voteCount": 27, "position": 2 }
  ]
}

Reglas: requiere edicion con voting_open = true. Ejecutar cambio en transaccion y emitir evento WebSocket.

GET /rankings/public?editionSlug=edicion-2026

Ranking publico visible para todos.

Admin

EndpointUso
GET /admin/usersActual. Listar usuarios. Rol: admin.
PATCH /admin/users/:id/roleActual. Asignar rol. Body: { "role": "admin|exhibitor|jury|guest" }.
POST /admin/exhibitorsActual. Dar de alta expositor/equipo. Crea o actualiza usuario expositor y equipo asociado a una edicion.
POST /admin/editionsPlanificado. Crear edicion.
PATCH /admin/editions/:idPlanificado. Editar estado, fechas y ventana de votacion.
GET /admin/results?editionId=uuidPlanificado. Resultados publicos y de jurado.

Archivos

POST /files/presign

Genera URL firmada para subir archivos a MinIO. MinIO usa API compatible con S3.

{
  "editionId": "uuid",
  "projectId": "uuid",
  "kind": "logo",
  "fileName": "logo.png",
  "mimeType": "image/png",
  "sizeBytes": 120000
}
{
  "fileId": "uuid",
  "uploadUrl": "https://minio.uadestartup.example.com/...",
  "objectKey": "editions/2026/projects/uuid/logo.png"
}

Valores esperados de kind: logo, image, video, deck u attachment. El campo videoUrl de proyectos queda reservado para videos externos; los videos subidos por la app se manejan como archivos en MinIO.

Realtime

WS /realtime

Canal WebSocket para eventos de ranking publico.

{
  "type": "ranking.updated",
  "editionId": "uuid",
  "changedProjects": [
    { "projectId": "uuid", "voteCount": 28, "position": 1 },
    { "projectId": "uuid", "voteCount": 27, "position": 2 }
  ]
}

El cliente Next.js debe actualizar el feed sin refrescar la pagina cuando recibe ranking.updated.
