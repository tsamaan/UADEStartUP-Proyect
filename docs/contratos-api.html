Contratos de API - UADE StartUP

Contratos REST + WebSocket para el MVP. Los endpoints de auth interna, ediciones, admin y proyectos reflejan el backend Express actual; votacion, jurado, archivos y realtime quedan como contratos planificados.

Convenciones

Base URL actual/. El backend actual expone rutas sin prefijo /api/v1.
FormatoJSON.
Auth internaSesion HTTP-only o Bearer token para admin, jurado y expositor.
Auth invitadosNo se requiere login para ver ediciones, proyectos y ranking publico. Google OAuth solo es obligatorio para votar.
Errores{ "error": { "code": "string", "message": "string", "details": {} } }

GET /auth/google

Planificado. Redirecciona al flujo OAuth Google. Se dispara cuando un invitado intenta votar sin estar autenticado.

GET /auth/google/callback

Planificado. Callback OAuth. Crea o recupera usuario invitado, valida que use cuenta Google y lo deja autenticado.

POST /projects

Crea proyecto. Rol: expositor o admin.

{
  "editionId": "uuid",
  "teamId": "uuid",
  "title": "Proyecto Demo",
  "shortSummary": "Resumen corto",
  "description": "Descripcion completa",
  "problem": "Problema",
  "solution": "Solucion",
  "category": "Categoria",
  "stage": "mvp",
  "contactEmail": "equipo@uade.edu.ar",
  "demoUrl": "https://example.com/demo",
  "repositoryUrl": "https://github.com/org/repo"
}

Logo, imagenes y videos no se envian como links en este body. Se suben con POST /files/presign usando kind: logo, image o video.

PATCH /projects/:id

Edita proyecto. Rol: expositor propietario o admin.

POST /projects/:id/submit

Envia proyecto a revision.

POST /projects/:id/publish

Publica proyecto. Rol: admin.

Votos

POST /projects/:id/vote

Actual. Emite o cambia el voto activo del invitado autenticado con Google para la edicion del proyecto. Requiere Google Login.

{
  "previousProjectId": "uuid-or-null",
  "currentProjectId": "uuid",
  "ranking": [
    { "projectId": "uuid", "voteCount": 28, "position": 1 },
    { "projectId": "uuid", "voteCount": 27, "position": 2 }
  ]
}

Reglas: requiere edicion con voting_open = true. Ejecutar cambio en transaccion y emitir evento WebSocket.

Archivos

POST /files/presign

Actual. Genera URL firmada para subir archivos a MinIO. Rol: admin o expositor propietario del proyecto.

{
  "projectId": "uuid",
  "kind": "logo",
  "fileName": "logo.png",
  "mimeType": "image/png",
  "sizeBytes": 120000
}
{
  "file": {
    "id": "uuid",
    "kind": "logo",
    "originalName": "logo.png",
    "bucket": "uade-startup",
    "objectKey": "editions/edicion-2026-primer-cuatrimestre/projects/uuid/logo/uuid-logo.png",
    "publicUrl": "http://localhost:9000/uade-startup/editions/...",
    "mimeType": "image/png",
    "sizeBytes": 120000
  },
  "upload": {
    "method": "PUT",
    "url": "https://minio.uadestartup.example.com/...",
    "expiresInSeconds": 900,
    "headers": {
      "Content-Type": "image/png"
    }
  }
}

Valores esperados de kind: logo, image, video, deck u attachment. El cliente sube el binario con PUT a upload.url. Logo, imagenes y videos del proyecto deben cargarse por este flujo; no se aceptan links externos para esos medios.

Realtime

Socket.IO /realtime

Actual. Canal Socket.IO para eventos de ranking publico.

Al conectarse, el cliente debe unirse a una sala de edicion:

{
  "event": "edition.join",
  "payload": {
    "editionId": "uuid"
  }
}

Cuando cambia un voto, el servidor emite:

{
  "type": "ranking.updated",
  "editionId": "uuid",
  "changedProjects": [
    { "projectId": "uuid", "voteCount": 28, "position": 1 },
    { "projectId": "uuid", "voteCount": 27, "position": 2 }
  ],
  "ranking": [
    { "id": "uuid", "title": "Proyecto Demo", "voteCount": 28, "position": 1 }
  ]
}

El cliente Next.js debe actualizar el feed sin refrescar la pagina cuando recibe ranking.updated.
