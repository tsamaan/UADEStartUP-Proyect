POST /projects — campo members (extensión documentada)

El payload de creación de proyecto acepta un campo adicional members que representa los integrantes del equipo. Este campo se persiste en teams.members (tipo jsonb) según el modelo entidad-relación.

Este campo fue agregado al formulario de frontend en la pantalla /panel/expositor (componente CargarProyectoForm.tsx). El backend debe aceptarlo en el body del POST /projects o en un endpoint separado PATCH /teams/:id para actualizar los integrantes. 

{
  "editionId": "uuid",
  "teamId": "uuid",
  "title": "Proyecto Demo",
  "shortSummary": "Resumen corto",
  "description": "Descripcion completa",
  "problem": "Problema",
  "solution": "Solucion",
  "category": "Tecnologia",
  "stage": "mvp",
  "contactEmail": "equipo@uade.edu.ar",
  "demoUrl": "https://demo.miproyecto.com",
  "repositoryUrl": "https://github.com/org/repo",
  "members": [
    { "name": "Juan Garcia", "legajo": "12345" },
    { "name": "Maria Lopez", "legajo": "67890" }
  ]
}

Cada integrante tiene: name (string, nombre y apellido completo) y legajo (string, número de legajo universitario). El array puede contener 1 o más integrantes sin límite fijo.

Estado: el frontend ya envía este campo. El backend debe mapearlo a teams.members al crear o actualizar el equipo. Pendiente validación en backend Sprint correspondiente.
