Roadmap para construir el backend en Node.js con Express, PostgreSQL en CapRover, MinIO, Google Login para votar y WebSockets para ranking en vivo.

Estado actualizado: Sprint 1 y Sprint 2 completos. Prisma, PostgreSQL local, migracion inicial, seed y endpoints publicos de ediciones ya fueron verificados contra la base real.

Criterio general

El backend se construye de menor a mayor riesgo: base del proyecto, datos, permisos, proyectos, votos, tiempo real, jurado, archivos y deploy. Cada sprint debe cerrar con algo ejecutable y verificable.

Orden recomendado: no arrancar por Google OAuth, WebSockets o MinIO. Son piezas importantes, pero necesitan una base estable para no volverse barro.
