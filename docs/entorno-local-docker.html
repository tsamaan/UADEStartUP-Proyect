Entorno local Docker - UADE StartUP

Guia para levantar UADE StartUP en cualquier PC usando este repo de infraestructura.

Objetivo

Este repositorio no esta pensado para desplegarse en CapRover. Su objetivo es permitir que una persona clone los repos del proyecto y levante un entorno local reproducible con Docker.

Estructura esperada

proyectosTeo/
  UADEStartUP-Proyect/   # infra + docs
  UADEStartUP-Backend/   # backend Express
  UADEStartUP-Frontend/  # frontend Next.js

Por defecto, el Compose busca el backend en ../UADEStartUP-Backend y el frontend en ../UADEStartUP-Frontend. Si estan en otra ruta, cambiar BACKEND_PATH o FRONTEND_PATH en .env. Si se usan los scripts con -WithBackend, -WithFrontend o -WithApp y alguno de los repos no existe, se clona automaticamente desde su URL configurada.

Primer uso

  1. Entrar al repo de infra.
  2. Copiar variables de ejemplo.
  3. Levantar servicios base.
  4. Levantar backend y frontend.
cd C:\Users\bond\Desktop\proyectosTeo\UADEStartUP-Proyect
Copy-Item .env.example .env
docker compose up -d postgres minio minio-init
docker compose --profile app up backend frontend

Tambien se pueden usar los scripts incluidos. Las variantes -WithBackend y -WithFrontend verifican si existen los repos correspondientes y los clonan automaticamente si faltan. La variante -WithApp levanta backend y frontend juntos.

.\scripts\dev-up.ps1
.\scripts\dev-up.ps1 -WithBackend
.\scripts\dev-up.ps1 -WithFrontend
.\scripts\dev-up.ps1 -WithApp

Puertos ocupados

Si ya hay otro PostgreSQL, MinIO, backend o frontend corriendo en la PC, puede haber conflicto de puertos. Hay dos caminos:

  • Apagar el entorno anterior con docker compose down desde el repo que lo levanto.
  • Cambiar puertos en .env, por ejemplo POSTGRES_PORT=5433, BACKEND_PORT=3002 o FRONTEND_PORT=3003.
Tip: si se usa este repo de infra como entrada principal, conviene levantar PostgreSQL y MinIO solo desde aca para evitar duplicados.

Datos persistentes

PostgreSQL y MinIO usan volumenes Docker. Los datos sobreviven a docker compose down, pero se borran con docker compose down -v.

Atencion: este entorno es para desarrollo local. Las credenciales de .env.example no deben usarse como secretos reales de produccion.

Frontend

El servicio frontend levanta Next.js desde el repo UADEStartUP-Frontend. Por defecto expone la app en http://localhost:3001 y configura NEXT_PUBLIC_API_URL=http://localhost:3000 para consumir el backend local.
